## Task 3: Supabase Schema

Run this SQL in the Supabase SQL editor:

```sql
-- Threads table: maps our user to an OpenAI thread
CREATE TABLE threads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  openai_thread_id TEXT NOT NULL UNIQUE,
  title TEXT NOT NULL DEFAULT 'New conversation',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Row Level Security
ALTER TABLE threads ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users see own threads" ON threads
  FOR ALL USING (auth.uid() = user_id);
```

**Validation:** Insert a row with a fake user_id, confirm RLS blocks reads from a different user context.

## Task 4: Auth Middleware (Backend)

### `auth.py`
Create a FastAPI dependency `get_current_user` that:
1. Extracts the `Authorization: Bearer <token>` header
2. Calls `supabase.auth.get_user(token)` to verify the JWT
3. Returns the user object, or raises `HTTPException(401)` on failure

All protected routes use `Depends(get_current_user)`.

**Validation:** Calling a protected endpoint without a token returns 401.

## Task 5: Thread Endpoints

### `routers/threads.py`

```
POST   /api/threads       - Create a new thread (calls OpenAI to create thread, saves to DB)
GET    /api/threads       - List user's threads (from DB, newest first)
DELETE /api/threads/{id}  - Delete thread (delete from DB; OpenAI thread is abandoned)
PATCH  /api/threads/{id}  - Update thread title
```

**Thread creation flow:**
1. Call `openai.beta.threads.create()` → get `openai_thread_id`
2. Insert row into `threads` table with `user_id` from JWT
3. Return the thread record

**Validation:**
- POST /api/threads with valid JWT → returns thread with `openai_thread_id`
- GET /api/threads → returns list of user's threads
- Confirm thread appears in Supabase dashboard

## Task 6: Chat Streaming Endpoint

### `services/openai_service.py`

Wrap the OpenAI Responses API calls:

```python
async def stream_chat(thread_id: str, message: str) -> AsyncGenerator[str, None]:
    # 1. Add user message to the thread
    await openai.beta.threads.messages.create(
        thread_id=thread_id,
        role="user",
        content=message
    )
    # 2. Create a streaming run
    async with openai.beta.threads.runs.stream(
        thread_id=thread_id,
        assistant_id=settings.openai_assistant_id,
    ) as stream:
        async for text in stream.text_deltas:
            yield text
```

### `routers/chat.py`

```
POST /api/chat/stream
Body: { thread_id: string, message: string }
Response: text/event-stream (SSE)
```

**SSE format:**
```
data: <text chunk>\n\n
data: [DONE]\n\n
```

Use FastAPI's `StreamingResponse` with `media_type="text/event-stream"`.

**Validation:**
- Send a message to a thread → streaming SSE response arrives
- Full response is coherent
- `[DONE]` event signals completion

## Task 10: Frontend Chat UI

### `hooks/useThreads.ts`
- Fetch threads from `GET /api/threads` on mount (with JWT from Supabase session)
- `createThread()` → POST /api/threads → prepend to list → select it
- `deleteThread(id)` → DELETE /api/threads/:id → remove from list

### `ThreadSidebar.tsx`
- "New conversation" button at top
- List of threads (title + relative date)
- Highlight active thread
- Delete button (on hover)

### `ChatWindow.tsx`
State: `messages: Message[]`, `isStreaming: boolean`, `input: string`

**Send message flow:**
1. Append user message to local state
2. POST to `/api/chat/stream` with `{ thread_id, message }`
3. Read the SSE stream: append assistant message, update content on each `data:` event
4. On `[DONE]` → set `isStreaming = false`

**Auto-title:** After first message, PATCH thread title to first 50 chars of user message.

### `MessageBubble.tsx`
- User messages: right-aligned, primary color
- Assistant messages: left-aligned, muted background
- Show a blinking cursor while streaming

**Validation:**
- Type a message → it appears in the chat
- Assistant response streams in token by token
- Starting a new conversation creates a new thread
- Switching threads loads no history (Module 1: OpenAI manages state, we don't load messages from API — that comes in Module 2)
- Page refresh keeps the thread list
